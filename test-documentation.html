🧪 Orange HRM Test Automation

Comprehensive Test Automation Project Documentation

📋 Project Overview

This project demonstrates automated testing for the Orange HRM application using Playwright and the Page Object Model (POM) design pattern. The test suite validates the complete workflow of managing employee dependents, from login to logout.

🎯 Key Objectives:
  • Automate critical user workflows in Orange HRM
  • Implement Page Object Model for maintainability
  • Ensure data consistency and test reliability
  • Generate comprehensive test reports

🏗️ Page Object Model Architecture

The Page Object Model (POM) is a design pattern that improves test maintainability by abstracting page elements and actions into reusable classes.

📝

LoginPage

Handles authentication flows - locators for username, password fields and login button. Methods: navigate(), login(), getErrorMessage()

🏠

DashboardPage

Manages dashboard navigation and user interactions. Methods: waitForDashboard(), navigateToMyInfo(), logout()

👥

DependentsPage

Manages dependent records - add, view, verify, delete operations. Methods: addDependent(), isDependentPresent(), getDependentCount()

1

Open Login Page

Navigate to Orange HRM login page at https://opensource-demo.orangehrmlive.com/web/index.php/auth/login. Wait for branding logo to ensure page is fully loaded.

2

Login with Valid Credentials

3

Navigate to My Info

Click "My Info" in the left sidebar. Application redirects to personal details page. Wait for URL change to verify navigation.

4

Click Dependents Tab

Click "Dependents" tab under My Info section. Wait for URL to contain "viewDependents" and "Add" button to be visible.

5

Add a New Dependent

Click "Add" button. Fill in dependent details: Name, Relationship (Child/Spouse/Other), and Date of Birth. Click Save and wait for success notification.

Verify Dependent Addition

Confirm that the newly added dependent appears in the dependents table. Verify table row count increased and dependent name is visible.

7

Logout

Click user dropdown menu and select logout. Verify redirect to login page. Test suite completes successfully.

🚀 Running the Tests

Run Tests

# Run tests with browser UI visible
npm run test:headed

# Debug mode with step-by-step execution
npm run test:debug
View Test Report
npx playwright show-report

💡 Best Practices Implemented

  • Page Object Model (POM): Separates test logic from UI element locators for better maintainability
  • Explicit Waits: Uses proper wait conditions instead of hard timeouts
  • Test Data Management: Centralized test data in utils/testData.js for easy updates
  • Error Handling: Proper assertions and error messages for debugging
  • Screenshots & Videos: Automatic capture on test failure for debugging
  • Serial Execution: Tests run one at a time for consistent state management
  • HTML Reporting: Comprehensive test reports with execution details

🔧 Troubleshooting Guide

Common Issues & Solutions
  • Locator Ambiguity Error: Multiple elements match the selector. Solution: Use more specific locators or add context filters.
  • Timeout Errors: Page element not found within timeout. Solution: Increase timeout or verify element visibility conditions.
  • Network Issues: Cannot reach OrangeHRM. Solution: Check internet connection and baseURL in playwright.config.js.
  • Test Flakiness: Tests pass/fail inconsistently. Solution: Add proper waits and verify test isolation.
